+# راهنمای سریع ابزارها
+
+این پروژه دو مرحله اصلی دارد: ساخت دیتاست مصنوعی برای YOLO و گرفتن خروجی شمارش روی تصاویر جدید. دستورات زیر را در پوشه ریشه‌ی مخزن اجرا کنید.
+
+## 1) ساخت دیتاست مصنوعی شبیه عکس مشتری
+```bash
+python tools/synthetic-data-generator.py \
+  -n 120 \                # تعداد عکس مصنوعی
+  -g 250 -G 850 \          # بازه‌ی تعداد گرافت در هر عکس
+  -o synthetic_dataset \   # مسیر خروجی
+  --bg-variants 3 \        # تنوع نور/رنگ پس‌زمینه
+  --overlap 0.4 \          # چسبندگی/روی‌هم‌افتادگی گرافت‌ها
+  --blur 0.35 \            # بلور تصادفی برای فوکوس‌های مختلف
+  --val-ratio 0.1 \        # نسبت اعتبارسنجی (10٪)
+  --seed 42                # تکرارپذیری
+```
+خروجی شامل پوشه‌های `images/train|val`، `labels/train|val`، فایل `data.yaml` (برای YOLO) و `manifest.json` (برای بررسی سریع تعداد گرافت‌ها) است.
+
+## 2) آموزش YOLO روی دیتاست ساخته‌شده
+دستور نمونه با پکیج `ultralytics` (نیاز به GPU برای سرعت بهتر):
+```bash
+yolo detect train data=synthetic_dataset/data.yaml model=yolov8n.pt epochs=100 imgsz=1024
+```
+پس از آموزش، وزن‌های نهایی در مسیر `runs/detect/train*/weights/best.pt` ذخیره می‌شود. مسیر مدل را در متغیر محیطی `YOLO_MODEL` یا تنظیمات اپ وارد کنید.
+
+## 3) تست شمارش روی عکس‌های واقعی/جدید
+از اسکریپت CLI آماده برای تولید خروجی و CSV استفاده کنید:
+```bash
+python tools/cli_count.py "sample_images/*.jpg" -o outputs --csv counts.csv
+```
+در صورت وجود مدل YOLO، به‌طور خودکار استفاده می‌شود؛ در غیر این صورت، الگوریتم کلاسیک (CV) فعال می‌شود. خروجی شامل تصویرهای دارای نقطه‌گذاری و فایل CSV تعداد هر عکس است.
+
+## نکات دقت ~۹۰٪
+- قبل از آموزش، چند نمونه از `images/` را بازبینی کنید و در صورت نیاز برچسب‌ها را دستی اصلاح کنید.
+- اگر نور ظرف مشتری متفاوت است، مقدار `--bg-variants` و `--overlap` را کمی بیشتر کنید تا مدل به تنوع بیشتر عادت کند.
+- زمان تست، ظرف پتری را کامل در کادر بگذارید تا ماسک خودکار ظرف نقاط بیرونی را حذف کند.
